# CS 305 Portfolio – Artemis Financial Security Assessment

## Client Summary

Artemis Financial is a financial consulting company that develops personalised financial plans involving investments, retirement planning, savings, and insurance services. The company sought to modernise its software systems while strengthening software security protections surrounding client financial information and secure communications.

The primary issue addressed during this project involved identifying software security vulnerabilities within Artemis Financial’s web application and recommending mitigation strategies aligned with contemporary secure software development standards.

## Security Vulnerability Assessment

One aspect performed effectively throughout the vulnerability assessment process involved analysing security weaknesses across multiple layers of the application architecture. This included reviewing cryptographic implementations, client-server communications, APIs, input validation, and dependency vulnerabilities. Conducting static testing and manually reviewing code structures significantly improved understanding of secure development practices.

Secure coding remains critically important because vulnerabilities within financial systems may expose confidential customer information, compromise secure communications, and damage organisational credibility. Strong software security protections help preserve customer trust, regulatory compliance, operational stability, and long-term business integrity.

## Challenges and Learning Experience

One of the more challenging aspects of the assessment process involved interpreting dependency-check vulnerability reports and understanding mitigation strategies associated with third-party libraries. However, this also became one of the most valuable learning experiences because it demonstrated how external dependencies may introduce security risks even when internally written code appears secure.

The vulnerability assessment process flow diagram greatly assisted in understanding how security concerns are distributed across architecture reviews, cryptography, APIs, secure coding practices, and encapsulation layers.

## Security Enhancements and Mitigation

Security layers were increased by analysing secure communication protocols, evaluating cryptographic standards such as SHA-256, reviewing dependency vulnerabilities, and recommending HTTPS implementation practices alongside secure certificate handling.

In future projects, tools such as OWASP Dependency Check, static testing frameworks, vulnerability scanners, and secure code review methodologies would continue to be utilised when assessing risks and selecting mitigation techniques.

## Functional and Secure Code Validation

After refactoring and reviewing the software application, testing procedures were performed to ensure the application remained functional while reducing security vulnerabilities. Static testing reports, dependency analysis, and manual code inspections were used to confirm that additional vulnerabilities were not introduced during remediation efforts.
